# Stakeholder Analysis – Smart Building IoT System

---

## 1. Project Stakeholders Overview

The Smart Building IoT System involves multiple stakeholders who interact with the system for monitoring, controlling, and managing building operations.

---

## 2. Key Stakeholders

### 1. Building Owner
- Owns the building and invests in the system
- Interested in cost reduction and ROI

### 2. Facility Manager
- Manages daily building operations
- Monitors energy usage, security, and system performance

### 3. Security Personnel
- Monitors security alerts and CCTV integration
- Responds to intrusion or emergency alerts

### 4. Maintenance Team
- Handles device maintenance and repair
- Ensures all IoT devices are functioning properly

### 5. System Administrator
- Manages system configuration and user access
- Controls backend settings and integrations

### 6. Tenants / Occupants
- Use building facilities
- Benefit from automation (lighting, temperature control)

### 7. IoT Devices (Sensors & Controllers)
- Environmental sensors (temperature, humidity, motion)
- Smart devices (lights, AC, alarms, locks)

---

## 3. Stakeholder Expectations

| Stakeholder | Expectations |
|-------------|-------------|
| Building Owner | Cost reduction, ROI, efficiency |
| Facility Manager | Real-time monitoring, control |
| Security Personnel | Instant alerts, safety |
| Maintenance Team | Easy diagnostics, alerts |
| System Admin | Secure system control |
| Tenants | Comfort and automation |

---

## 4. Stakeholder Communication Needs

- Real-time alerts via mobile/web dashboard
- Weekly performance reports
- Incident notifications
- System status dashboards
